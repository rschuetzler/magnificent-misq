  <action>
Create `misq.typ` at project root with a `#let misq(...)` function following the amcis.typ pattern. The function must include:

**Function signature** (define all params upfront per user decision, even if not all functional in Phase 1):
- `title: [Untitled]` — content
- `abstract: none` — content or none
- `keywords: ()` — array of strings, template formats with commas
- `bib: none` — string path or none, template handles #bibliography() internally
- `body` — positional, required, must be last parameter

**Page geometry** (PAGE-01, PAGE-02):
- `set page(paper: "us-letter", margin: (x: 1in, top: 1in, bottom: 1in))`
- This yields 6.5" text width and 9" text height matching the LaTeX template

**Font** (TYPO-01):
- `set text(font: ("Times New Roman", "Times"), size: 12pt)`
- Add code comment: "Times New Roman is required by MISQ; Times is the macOS/Linux fallback"
- Do NOT bundle font files. Do NOT add runtime font checks. Silent fallback per user decision.

**Body spacing** (TYPO-02):
- `set par(justify: true, leading: 0.85em, spacing: 0.85em)` as starting estimate for double-spacing
- Both `leading` AND `spacing` must be set (leading = intra-paragraph, spacing = inter-paragraph)
- Add comment noting these are calibrated estimates; Typst leading is gap-based not baseline-based
- Use inline values, NOT named constants (per user decision)

**Abstract spacing** (TYPO-03):
- When rendering abstract content, wrap in a block with scoped par settings:
  `block({ set par(leading: 0.55em, spacing: 0.55em); abstract })` as 1.5x starting estimate

**Single-spacing regions** (TYPO-04, TYPO-05):
- `show bibliography: set par(leading: 0.65em, spacing: 0.65em)` for references
- `show figure: set par(leading: 0.65em)` for figures
- `show table: set par(leading: 0.65em)` for tables
- Use show rules, NOT helper wrapper functions (per user decision)

**Front matter rendering** (minimal for Phase 1 — structure comes in Phase 2):
- If abstract is not none, render it with "Abstract" label and 1.5x spacing block
- If keywords is not empty, render bold "Keywords:" followed by comma-joined keywords
- Render body content
- If bib is not none, call `bibliography(bib, style: "apa")` (the bibliography call itself, not just passing through)

**File structure:**
- Move `references.bib` from project root to `template/references.bib` (use git mv if possible, otherwise copy and delete)
- Create `template/main.typ` — a minimal test document that:
  - Imports misq from the parent directory: `#import "../misq.typ": misq`
  - Calls `#show: misq.with(title: [...], abstract: [...], keywords: (...), bib: "references.bib")`
  - Includes 2-3 paragraphs of body text (enough to verify double-spacing across lines)
  - Includes a sample figure with a caption (even just a rect placeholder) to test figure single-spacing
  - Includes a sample table with a caption to test table single-spacing
  - References the sample bib entries to test bibliography single-spacing

**Anti-patterns to avoid:**
- Do NOT define named spacing constants (e.g., `let body_leading = 0.85em`) — use inline values per user decision
- Do NOT create helper wrapper functions for spacing — use show rules per user decision
- Do NOT use `set text(font: "serif")` — Typst does not support generic family names
- Do NOT use deprecated `show par: set block(spacing:)` syntax — use `set par(spacing:)` directly

**Step 1 — Generate LaTeX reference PDF:**
Run `pdflatex MISQ_Submission_Template_LaTeX.tex` in the project root to produce a reference PDF for visual comparison. If pdflatex is not available, skip this step and use the known LaTeX baselinestretch values as the reference instead.

**Step 2 — Verify font embedding (TYPO-01):**
Run `pdffonts template/main.pdf` (or equivalent) to confirm Times New Roman or Times is embedded in the PDF. If the font shows as "Libertinus Serif" or similar, the system does not have TNR installed — this is acceptable per user decision (silent fallback), but note it in the summary.

**Step 3 — Calibrate body double-spacing (TYPO-02):**
Open both PDFs (LaTeX and Typst). Compare the body text line spacing visually. The Typst output should look "visually equivalent" to the LaTeX double-spaced output — not pixel-perfect, but the same general density and feel. If the spacing is noticeably tighter or looser than the LaTeX output:
- Too tight: increase `leading` and `spacing` values in the body `set par()` call
- Too loose: decrease them
- Adjust in increments of 0.05em, recompile, and compare again
- The goal is visual equivalence, not mathematical precision

**Step 4 — Calibrate abstract 1.5x spacing (TYPO-03):**
Compare the abstract region. The Typst abstract should look like 1.5x spacing compared to the body's 2x. Adjust the `leading` and `spacing` values in the abstract block wrapper if needed.

**Step 5 — Verify single-spacing regions (TYPO-04, TYPO-05):**
Confirm that bibliography entries, figure content, and table content appear single-spaced (noticeably tighter than body text). The default Typst leading of 0.65em should produce this effect.

**Step 6 — Update misq.typ with calibrated values:**
Replace the starting estimate values with the calibrated ones. Add comments noting the final values and that they were calibrated against the LaTeX reference output.

After calibration, do a final compile of `template/main.typ` and confirm no errors.
