<task type="auto">
  <name>Task 2: Add heading show rules with numbered styling</name>
  <files>misq.typ</files>
  <action>
Add heading configuration and per-level show rules inside the misq() function, after the page/font/spacing setup but before the front-matter rendering:

1. **Global heading numbering (HEAD-04):**
   ```typst
   set heading(numbering: "1.1.")
   ```
   Note: The trailing period in the numbering pattern may produce "1." rendered headings. If the user decision says "1, 2, 3" without periods, use a custom numbering function that strips the trailing period. Test with `"1.1"` (no trailing dot) first — Typst may handle this correctly for all levels.

2. **Level 1 show rule (HEAD-01):** Centered, uppercase, bold, 12pt, numbered
   ```typst
   show heading.where(level: 1): it => block(
     above: 1.4em,
     below: 0.8em,
     {
       set align(center)
       set text(weight: "bold", size: 12pt)
       if it.numbering != none {
         counter(heading).display(it.numbering)
         [ ]
       }
       upper(it.body)
     }
   )
   ```

3. **Level 2 show rule (HEAD-02):** Centered, bold, 12pt, numbered (no uppercase)
   ```typst
   show heading.where(level: 2): it => block(
     above: 1.4em,
     below: 0.8em,
     {
       set align(center)
       set text(weight: "bold", size: 12pt)
       if it.numbering != none {
         counter(heading).display(it.numbering)
         [ ]
       }
       it.body
     }
   )
   ```

4. **Level 3 show rule (HEAD-03):** Left-aligned, bold, 12pt, numbered
   ```typst
   show heading.where(level: 3): it => block(
     above: 1.4em,
     below: 0.8em,
     {
       set align(left)
       set text(weight: "bold", size: 12pt)
       if it.numbering != none {
         counter(heading).display(it.numbering)
         [ ]
       }
       it.body
     }
   )
   ```

Spacing values (above: 1.4em, below: 0.8em) are uniform across all levels per user decision. These are starting values — visual equivalence to LaTeX is the goal, not exact matching. The LaTeX template uses -3.5ex/+2.3ex for sections and -3.25ex/+1.5ex for subsections, but user decided on uniform spacing.

IMPORTANT: The `counter(heading).display(it.numbering)` pattern is critical — using `it.body` alone in a transformational show rule loses the heading number. See RESEARCH.md Pitfall 1.
