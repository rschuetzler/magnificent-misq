<task type="auto">
  <name>Task 1: Create Makefile with publish automation</name>
  <files>Makefile</files>
  <action>
Create Makefile at the project root with the following structure:

**Config block at top (portability — per DEPLOY-03):**
```makefile
# ============================================================
# Configuration — change these lines for another project
# ============================================================
PKG_NAME    := misq
GITHUB_USER := rschuetzler
FORK_REPO   := https://github.com/$(GITHUB_USER)/typst-packages.git

# Derived — do not edit
PKG_VERSION := $(shell grep '^version' typst.toml | sed 's/version = "\(.*\)"/\1/')
BRANCH_NAME := $(PKG_NAME)-$(PKG_VERSION)
PKG_DIR     := packages/preview/$(PKG_NAME)/$(PKG_VERSION)

# Files to include in the package (explicit list)
PKG_FILES   := misq.typ apa.csl typst.toml thumbnail.png LICENSE README.md
PKG_DIRS    := template
```

**Targets:**

1. `.PHONY: publish thumbnail clean help`

2. `help` (default target): Print available targets using grep/awk on Makefile comments (## pattern). This is the default target so bare `make` shows help.

3. `thumbnail`: Generate thumbnail.png from template first page.
   ```
   typst compile --format png --ppi 250 --pages 1 template/main.typ thumbnail.png
   ```
   Print confirmation with file size.

4. `publish`: Full publish workflow. Depends on `thumbnail` target. The target body is a single bash script block (use `.ONESHELL:` or chain with `&&` and backslash continuations). Steps:
   - Check that `typst` CLI is available (command -v typst), error with message if not
   - Check that `git` is available
   - Print PKG_NAME, PKG_VERSION, BRANCH_NAME for user confirmation
   - Create temp directory with mktemp -d
   - Sparse clone the fork: `git clone --filter=blob:none --sparse --depth 1 "$(FORK_REPO)" "$$WORK_DIR"` (per DEPLOY-02)
   - cd into the clone, set sparse-checkout: `git sparse-checkout set "packages/preview/$(PKG_NAME)"`
   - Create branch: `git checkout -b "$(BRANCH_NAME)"`
   - Create package directory: `mkdir -p "$(PKG_DIR)"`
   - Copy each file in PKG_FILES to PKG_DIR
   - Copy template directory: `cp -r template/ "$(PKG_DIR)/template/"`  (note: do NOT copy template/.DS_Store — use `rsync -a --exclude '.DS_Store'` or copy individual files)
   - For template directory, use: `mkdir -p "$(PKG_DIR)/template" && cp template/main.typ template/references.bib "$(PKG_DIR)/template/"`  This avoids copying .DS_Store and other hidden files.
   - git add the PKG_DIR
   - git commit with message "Add $(PKG_NAME) $(PKG_VERSION)"
   - git push origin $(BRANCH_NAME)
   - Print the working directory path
   - Print a URL hint: "Create PR at: https://github.com/typst/packages/compare/main...$(GITHUB_USER):typst-packages:$(BRANCH_NAME)"

   Use `set -euo pipefail` at the start of the shell block for safety. Use `$$` for shell variables in Makefile (double dollar sign).

5. `clean`: Remove thumbnail.png only. Print what was removed.

**Important Makefile details:**
- Use tabs for indentation (required by Make)
- Use `SHELL := /bin/bash` at the top for bash features (set -euo pipefail)
- Use `.ONESHELL:` so the publish target runs as a single shell invocation
- Each recipe line that uses shell variables needs `$$VAR` syntax
- Add `## Description` comments after target names for the help target
